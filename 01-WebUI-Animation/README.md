# 模块一：动画与交互测试 (Web UI 层)

## 📌 模块定位
本文件夹（`01-WebUI-Animation`）用于隔离测试“记忆博物馆”的前端视觉交互体验。
核心目标是：检验在脱离后端的情况下，利用 Next.js 与动画库（如 Framer Motion），实现图片上传后的**粒子化交互效果**及**平滑视效表现**。

## 🎯 最终交付物
一个无需真实连接数据库即可完整预览视觉流的**高质感纯前端画廊 Demo**。

---

## 📋 模块任务拆解 (Task List)

1. **环境准备**
   - 在本目录下初始化一个新的 Next.js (App Router) 项目结构，但**精简不必要的 API 配置**，专注于页面渲染。
   - 引入当前的 `ui-reference` 代码（Vite 迁移至 Next.js 组件格式）。
2. **纯前端交互组件开发**
   - 开发“拖拽/点击上传多张图片”组件（仅在前端暂存图片，比如利用 FileReader 转成 base64）。
3. **粒子与显影特效集成**
   - 解析前端拿到的图片对象。
   - 接入已有的 Canvas / Framer Motion 效果：让图片从星砂状粒子逐渐聚拢显现。
4. **平滑转场与听觉反馈**
   - 图片在画廊中的 Hover 悬浮放大处理。
   - 翻页转场时的轻微音效与丝滑衔接。

---

## ⚠️ 模块开发规则 (Rules)
- **禁止连接数据库或后端！** 所有上传的图片通过浏览器内存（Blob URL 或 Base64）模拟返回。
- 文本内容（如日期、故事片段）用一段优雅的 JSON 假数据写死，重点看其**随粒子显现的排版效果**。
- 所有 UI 组件应当抽取到 `components/` 文件夹下，保持高复用性，为模块四的大脑接管做准备。
