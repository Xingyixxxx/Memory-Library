# 模块三：数据库与简单后端联调

## 📌 模块定位
本文件夹（`03-Database-Backend`）专注于为“记忆博物馆”打造坚实的数据底座和极其简化的个人体验级权限系统。由于项目不上线公网用于盈利，后台应该只满足主人的上传需求。

## 🎯 最终交付物
一个以 Supabase 为核心，具备多图、多语音挂载能力的纯后端能力模块与相关定义。并且包含对应的脚本，可以通过一条命令对数据结构进行插入和查询测试。

---

## 📋 模块任务拆解 (Task List)

1. **项目建库与表结构设计 (Supabase)**
   - 在 Supabase 后台建表：建立 `hooks`（如果需要）、`entries`（事件集/回忆篇章的主表） 以及 `media_assets`（媒体素材附属表）。
   - 创建一对多外键关联（One-to-Many），确立每条 `media_assets` 的内容必然归属于同一个 `entries`（Entry ID）。
2. **建存储桶 (Storage)**
   - 在 Supabase 中创建图片和语音存储的 Bucket。
3. **身份权限简化 (Auth / RLS)**
   - 关闭繁琐的公共注册流程，只开通管理员账号。或使用极其简单的密码机制保护。
   - 在 Row Level Security（RLS 策略）上仅允许自己读写。
4. **客户端对接测试 (CRUD)**
   - 利用 `supabase-js` 编写两个纯净的测试脚本：
     - `uploadTest.js`（模拟插入一条带 3 张图和文字的记忆记录，并取得图片的远程链接存库）
     - `fetchTest.js`（直接通过查询 Entry，顺手把里面所有挂载的 assets 以及媒体按照排版顺序一次性返回出来）

---

## ⚠️ 模块开发规则 (Rules)
- **只关心数据的一致性**，无需前端渲染，确保能成功通过外键联表查询即可。
- 所有敏感的 supabase keys 必须放入 `.env` 并加到 `.gitignore` 中，严禁泄漏。
